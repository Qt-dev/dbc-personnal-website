For

The for loop has different implementation in Ruby according to the situation (each, until, etc).

This is the go-to tool in Javascript. i is the index of the loop that you're in.

for (var i=0; i<maximum; i++){
  //Your code here.
}

For..in

For..in

This iterates through a fixed number of objects.
Here, i is the index of the loop and we have it set to iterate through 0-5.

for i in 0..5
  #Your code here.
end

This iterates over the property of an object without taking the order into account.
Here, i is the identification of the property (name). If the object is an array, it will instead be the index.

for (var i in object){
  if (object.hasOwnProperty(i)){
    //Your code here
  }
}

While

While

This loop execute the code until a condition becomes false. It is often used with a boolean (use "break" to get out of the loop), or with a counter variable

count = 0 
while count < 5
  count++
  #Do something
end

Same as in Ruby, this loop execute the code until the argument becomes true. It is often used with a boolean (use "break" to get out of the loop), or with a counter variable

while(true){
  //Do something
  if (finish-test){
  break
  };
}

Begin..While

Do..While

There is no Do..While in ruby. Instead, you use begin...end while. The use is the same as a while loop, except that it is executed once before the first test.
It is not a proper "loop", but just a way to implement the exact same thing as in Javascript. It is more of "combining" a block notation of ruby (with the begin/end) with a condition to get out of it.

count = 0
begin
  count++
  #Do something
end while count < 5

Exactly the same as the while loop, except that this time, the loop is executed once before the first test at the "while". It is a commonly used loop in Javascript, and takes no ";" at the end.

do{
  //Do Something
  if (finish-test){
  break
  };
} while(true)

Until

...

This executes codes until a threshold is hit, and a condition becomes true.. You can think of it as the counterpart of the while loop.

counter = 0
until counter > 5
  #Do something
  counter++
end

In javascript, there is no until method. Instead, you can use a while statement to test the anti-condition. This example shows the same code as Ruby's example, in Javascript.

var counter=0
while(!(counter >= 5)){
  //Do Something
  counter++
}

Each

...

This executes a block of code on each rank of an array, or a range. Following the Ruby documentation, the block can be define with do .. end or {}. each_with_index is a sister of the each method that allows you to use the index of the current loop.

array = [1,2,3]
array.each do
  #Something
end

There is no each method in Javascript. Instead, use a for method.

var array = [1,2,3]
for(var i = 0; i < array.length; i++){
  //Do Something
}

Times

...

This loop executes code block a given amount of time. Following Ruby Documentation, the block can be defined either with do .. end or {}.
If you take a number from it (with the notation |variable| inside the block), that variable will take the number of the current loop.

5.times { |number| #Do Something with our number taking values of 0 to 4 }

There is no equivalent in Javascript. One of the good ways to implement the same code would be to use a for loop that makes the counter go backwards. Just like in this example, you set the variable high, iterates until it gets to 0, and remove one from it on every loop.

for(var i = 5; i > 0 ; i--){
  //Do Something
}
